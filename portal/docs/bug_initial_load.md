# 障害報告: ポータル初回ロード時の表示不完全

## 再現手順

`./portal.sh` を Mac から実行し、ブラウザが開いた直後の状態を確認する。

## 確認済みの症状

### 1. WSL アイコン（env バッジ）が表示されない
- ヘッダー右上に表示される WSL / Mac アイコンが、立ち上げ直後は表示されない。
- 別タブに切り替えたり Cmd+Shift+R をすると表示される場合がある。

### 2. STATUS タブがロードされていない
- HOME タブで開いた状態から STATUS タブをクリックしても、データが表示されない（空のまま）。

### 3. LOGS タブのログ一覧がクリックできない
- LOGS タブを開いてもログファイルの一覧が表示されず、クリックできない。
- 別タブに行って戻ると表示される。

### 4. LIBRARY タブのチャンネルチップが表示されない（Cmd+Shift+R 後）
- LIBRARY タブで Cmd+Shift+R（ハードリロード）をすると WSL アイコンは表示されるが、チャンネルチップが出ない。
- 別タブに切り替えてから LIBRARY タブに戻ると表示される。

## 期待する動作

`./portal.sh` でブラウザが開いた瞬間から、全タブのデータが揃っていること。タブの切り替えや手動リロードなしに：
- ヘッダーの WSL アイコンが表示されている
- STATUS タブを開けばすぐデータが見える
- LOGS タブを開けばすぐログ一覧が見える
- LIBRARY タブを開けばすぐチャンネルチップが見える

## 環境

- サーバー: WSL（Ubuntu）上の FastAPI + uvicorn（ポート 8080）
- クライアント: Mac ブラウザ（Tailscale 経由で WSL IP に接続）
- 起動スクリプト: `./portal.sh`（Mac から実行、WSL で tmux セッション `yt-portal` を起動）
